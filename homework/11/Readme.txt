#CURRENCY CONVERTER

####Арихитектура:
1) Data Provider(есть ли реальный поставщик данных?)
- Передает данные по курсам валют
- Получает данные

2) Server
2.1)Importer
- Получает данные по курсам валют и проверяет валидность данных (Check Integrity)
- Записывает или обновляет данные в базе(Transaction)
- Дополняет граф валют до полного(Import Algorithm)

2.2)CC
- Получает запросы от клиентов, обращается к базе и формирует ответ

3) Clients emulations
- N эмуляций * K запросов в минуту

####Алгоритм импорта:
- Данные от поставщика – это граф из n компонент связности,
- Каждую компоненту связности дополняем до полного ориентированного графа:
    -- Асимтотическая сложность = (n**2),
    -- Используем алгоритм обхода дерева в ширину.
- Если компоненты не связаны между собой, то объеденить в общий граф мы не можем,
- При получении данных и достройки полных компонент указываем время импорта и расчета.

####Схема таблицы:
Колонки: Cur1, Cur2, Rate, UploadTime.
Значения: Cur1=text, Cur2=text, Rate=integer, UploadTime=timestamp(дата и время)

####Технологический стэк:
